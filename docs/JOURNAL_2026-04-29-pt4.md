## Decisões arquitecturais

- **Frontend only:** zero mudanças backend. `tournament_number` e
  `all_players_actions` já vinham em `hands.list`.
- **Render condicional gracefully:** 3 cenários (GG completo / non-GG só TM /
  sem TM nada) sem branching complexo.
- **Mantém spread cópia (`[...tHands]`)** para não mutar arrays originais —
  princípio funcional.
- **Locale en-US:** coerência com app actual (`formatTableStack`). Refactor
  pt-PT global fica para sessão futura se quiseres.
- **TM clicável:** `stopPropagation` crítico para não conflituar com
  expand/collapse do grupo.
