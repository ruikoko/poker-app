## 5. VALIDAÇÃO RECEBIDO — o guardião (travão do Rui)

**Distinção crucial:** **ESPERADO** (offset que a app calcula, conferido contra a HH) ≠
**RECEBIDO** (offset para onde o watcher **realmente navegou**, log `[ALVO nav: offset N]`,
só existe DEPOIS de cada mão correr). **Só o RECEBIDO apanha** bugs de navegação do
watcher, pitch descalibrado e **packs stale**. A 4321 original é a lição (app sabia, o
robot foi para o BTN na mesma). **O guardião permanente valida o RECEBIDO, não só o
ESPERADO.**

**Resultado parcial:** 6/6 packs frescos → `RECEBIDO == ESPERADO == Hero`. 1 stale
apanhado (`6114702304` `BTN raise/4` vs esperado `CO/3`) — **exactamente** o que o
RECEBIDO serve para apanhar.
