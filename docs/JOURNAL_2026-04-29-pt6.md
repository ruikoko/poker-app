## Lições

1. **Audit superficial vs ground truth real**: o audit anterior (`validate_pipeline_2_pt6.py`) deu 0 swaps em 90 hands ao comparar `apa[player].cards ↔ hand_villains[player]`. Mas isso é **coerência interna** (ambos derivam do mesmo apa). Audit deep contra HH text raw (parsing `Seat X: hash showed [...]`) é único método para detectar swap real vs ground truth.
2. **Idempotência é invariante crítico**: `_enrich_hand_from_orphan_entry` foi escrito assumindo 1 chamada por hand, mas múltiplos ZIPs disparam re-execução. Sem idempotência guard, mappings divergem.
3. **Workarounds locais não substituem fix raiz**: o comentário em `mtt.py:117-120` (`_detect_vpip_players` faz bypass de `_build_anon_map`) provava bug conhecido mas só foi corrigido quando o efeito era visível em produção (Tech Debt #20).
4. **Wipe + repopulação real > restore parcial**: ciclo wipe-import-validar é mais limpo que tentar reproduzir bug em ambiente isolado, especialmente para detectar bugs cross-call (#21).
