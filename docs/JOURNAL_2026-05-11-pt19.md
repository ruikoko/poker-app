### Bloco 4 — Descoberta crítica (caso `Daily Hyper $80`)

Após deploy de B2, smoke real revelou falha estrutural: TS importado em backfill semanas antes + SS postada hoje = nenhum match no TIER 0 porque `start_time` do TS estava fora da janela `posted_at_hint [-12h, -30min]`.

**Diagnóstico:** TS é **autoritativo post-jogo**. Não precisa de janela temporal para discriminar — quando o nome bate e o `tournament_number` é literal do header, é match determinístico. Aplicar janela ao TIER 0 cria um falso-negativo estrutural.

**Implicação:** TIER 0 precisa de discriminador alternativo quando há múltiplas instâncias do mesmo nome (ex: `Daily Hyper $80` que corre todos os dias). Solução: usar `prize_pool` e `total_players` como discriminadores estritos (vêm do header da lobby SS, são únicos por instância de torneio).

Motivou B2.1.

### Bloco 7 — Backfill operacional `GTw → pos-nko`

Mudança de produto pelo Rui: descontinuar a tag HM3 `GTw`, adoptar `pos-nko` (originalmente canal Discord) como tag canónica para "posição em torneios não-KO". 7 PASSOS no workflow buffered-diff:

1. **PASSO 1 (dry-run)** — Bloqueio inicial de acesso: `railway login` expirou, `DATABASE_PUBLIC_URL` do serviço Postgres tinha password stale (consequência do incidente de Bloco 0, painel não sincronizado). Dry-run revelou:
   - 25 rows em `hands` com `'GTw' = ANY(hm3_tags)` (gate `played_at >= '2026-01-01'`).
   - Distribuição: 14 PokerStars, 1 Winamax, 10 WPN. **Zero GGPoker.**
   - 0 overlap com `pos-nko`.
   - 2 mãos têm `nota++` adicional (preservada).

2. **PASSO 2 (plano)** — discrepância semântica entre `hm3_tags` (lista canónica `HM3_REAL_TAGS` com ids do HM3) e `pos-nko` (canal Discord, sem id HM3). Decisão: id sintético `9999`, helper extraído em `services/hm3_tag_aliases.py` (opção β), 2 unit tests pure.

3. **PASSO 3 (rascunho commit)** — texto preparado em buffer antes de aplicar.

4. **PASSO 4 (backfill SQL)** — UPDATE atómico em prod via `psycopg2` com `autocommit=False`: snapshot count, UPDATE, e duas COUNTs de verificação no mesmo connection. Snapshot prévio salvo em `_local_only/backfill_GTw_20260511T225750Z.txt`. SQL final:
   ```sql
   UPDATE hands
      SET hm3_tags = ARRAY(
            SELECT DISTINCT t
              FROM unnest(array_replace(hm3_tags, 'GTw', 'pos-nko')) AS t
          )
    WHERE 'GTw' = ANY(hm3_tags)
      AND played_at >= '2026-01-01';
   ```
   Resultado: **25 rows updated**, GTw_after=0 ✓, pos-nko delta=+25 (5→30) ✓.
