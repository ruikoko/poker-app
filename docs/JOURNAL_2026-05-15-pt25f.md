### Deprecation fix `datetime.utcnow()` (commit `e18c8ff`)

Triggered pelo Rui apanhar `DeprecationWarning` na consola do `.bat` HM3 (linha 106 do `hm3_export.py` no PC do Rui). Investigação revelou 3 ocorrências no repo:
- `routers/hands.py:1326` (`viewed_at` na response)
- `routers/hands.py:1371` (`studied_at` em UPDATE psycopg2)
- `routers/hm3.py:756` (`cutoff_date` para filtro `days_back`)

Substituído por `datetime.now(timezone.utc).replace(tzinfo=None)` em todos. Pure deprecation fix, bit-for-bit preservado (o consumidor de `cutoff_date` em `hm3.py:786` compara contra `datetime.strptime(...)` que é naive — qualquer mistura aware/naive levantaria `TypeError`, daí `.replace(tzinfo=None)`). Suite 367 PASSED inalterada.

### Rotação de password user `rui@pokerapp.com` (sem commit; DB-only)

Single-shot via `UPDATE users SET password_hash = $2b$12$...` na DB Railway production, contornando a inexistência de feature change-password na app. Justificação: a password antiga (`MudaEsta123!`) ficou exposta em texto plano em `hm3_export.py`, em zips de partilha, e em briefings de sessões — devia ser tratada como comprometida.

Processo (script Python single-shot, sem persistir nada):
1. `secrets.token_urlsafe(18)` → 24 chars, alphabet `[A-Za-z0-9_-]` (sem aspas/backslash).
2. `bcrypt.hashpw(..., bcrypt.gensalt(rounds=12))` → mesmo algoritmo de `app.auth.hash_password`.
3. `UPDATE users SET password_hash = %s WHERE email = 'rui@pokerapp.com'` (id=1, único user).
4. Smoke HTTP contra prod: POST `/api/auth/login` com nova → 200 + session cookie. POST com antiga → 401.
5. Password mostrada **uma vez** no chat para o Rui copiar para gestor de passwords + `config_local.py`.

Sem alterações ao repo. Tech debt nova `#CHANGE-PASSWORD-FEATURE` adicionada (ver TECH_DEBTS).
