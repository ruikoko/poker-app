### Bloco 7 — 3 bugs do watcher descobertos via smoke real

Sucesso mecânico expôs 3 bugs funcionais no `hrc_watcher.exe` do Baltazar. **Todos exigem fonte Python** (não dá para corrigir externamente).

#### Bug A — Equity model fixo Malmuth-Harville (sem switch FGS)

Watcher escolhe sempre `Malmuth-Harville ICM` no equity model do HRC. **Não tem branch** para `Multi-Table FGS`, que é o modelo correcto para mãos longe da FT (mid-game MTT).

**Sintoma:** todas as mãos saídas do HRC têm equity ICM-aware FT-style, mesmo as que deveriam usar FGS. Solver dá EVs deslocados para mãos mid-MTT.

**Impacto:** mãos resolvidas pelo HRC têm valor científico questionável para todas excepto FTs.

#### Bug B — Max players estático em "in hand"

Watcher usa `get_player_count_from_hh()` (regex `^Seat \d+: \S+ \(\d+ in chips`) → conta **jogadores sentados na HH** (ex: 9 ou 8 na mesa).

**Problema:** o HRC precisa de saber **jogadores relevantes à decisão**, ou seja:
- Da última raise pré-decisão até hero
- Mais o nº de jogadores depois do hero que ainda têm acção
- Excluindo jogadores que já fizeram fold

Para uma situação tipo `UTG raise, MP fold, CO fold, BTN call, SB fold, BB → hero`, máx players relevantes = 3 (UTG, BTN, BB-hero), não 8.

**Impacto:** árvore do solver cresce com seats vazios → milhares de combos irrelevantes → tempo de cálculo explode + EV diluído.

#### Bug C — Nome do JS hardcoded → tree >20GB → OOM crash

Watcher carrega script HRC Pro com nome literal hardcoded:

```
mtt_advanced_20211029 - 2 flats + bb close action size open 2x - 3x bvb.js
```

Este script tem ranges **muito largos** para certos sizings + estilos de jogo. Em mãos com profundidade média/alta, o solver tree atinge **>20GB de RAM** e crasha o HRC. Vimos isso em 5+ mãos consecutivas no smoke.

**Solução de mitigação imediata** (sem recompilar): substituir o ficheiro `mtt_advanced_..._bvb.js` (com nome literal exacto) por uma versão mais tight do Rui. Mas isto força **1 script global** — o ideal seria escolher script via metadata.

**Solução final** (pt23+): config externa no watcher para nome do JS por tag/profundidade.
