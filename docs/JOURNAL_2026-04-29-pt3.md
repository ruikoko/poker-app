## Lições

1. **Análise prévia (quick scan) ANTES de implementação salva ~1-2h de discovery
   durante implementação.** Saber localizações exactas + esforço + edge cases
   acelera tudo. Especialmente útil em sessões de "limpeza" onde múltiplos itens
   independentes vão ser atacados.

2. **Decompor itens grandes em fases zero-risco + decisão estética separada.**
   #13 → #13a + #13b. Fase técnica (centralização) commitada com confiança;
   fase estética (paleta) com validação visual obrigatória. Reduz blast radius
   por commit.

3. **Aliases retrocompatíveis (`SITE_COLORS_X as SITE_COLORS`) permitem refactor
   incremental sem renomear callers.** Em #13a, 3 ficheiros tocaram só 2 linhas
   cada (apagar const + adicionar import com alias).

4. **Validação visual em prod por checklist explícito (9 testes em #13b) reduz
   dúvida no fim — "9/9 OK" é decisivo.** Especificar testes ANTES da
   validação evita "parece bom" subjectivo.

5. **Recovery pós-hibernação requer git log + ls do output esperado para
   confirmar persistência.** Hibernação cortou execução autónoma; nada do
   trabalho real ficou perdido (commits + push prévios). Lição: confiar em
   commits/push, não em estado intermédio em memória.
