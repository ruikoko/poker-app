# JOURNAL pt62 — porta de entrada do lobby por Intuitive Tables (pasta única, classificação por nome) (9 Junho 2026)

## Contexto
O Rui passou a usar UMA pasta única do Intuitive Tables (C:\Users\User\Desktop\Batmen\it) onde convivem SS de MESA e de LOBBY. Objetivo: mover a entrada dos lobbys para o IT, deixando a pasta de Capturas do Windows (LOBBY_DIR) como 2ª via manual. Discord #lobbys mantém-se.

## Realidade vs design anterior
- O appimport roteava por SUBPASTA (it -> table-ss, lobby -> lobbys). A pasta real do Rui NÃO tem subpastas: mesas e lobbys misturados.
- 1ª hipótese (prefixo `.exe` = lobby) REJEITADA: o update recente do IT mete o título da janela no nome, e há MESAS com prefixo `GGnet.exe` (ex.: `GGnet.exe-...Blinds 250 _ 500 - Table 24...`). Logo o `.exe` não distingue.

## Regra adotada (determinística, por NOME)
- MESA se o miolo do nome tem marcador de mesa: GG -> " - Blinds " ou " - Table "; Winamax -> "(#<dígitos>)". -> POST /api/table-ss/upload.
- Senão -> LOBBY -> POST /api/lobbys/upload, captured_at derivado do timestamp do nome (-YYYYMMDDHHMMSS-NN), fallback mtime.
- Sem essa cauda -> SKIP (legados "Shot..." ficam no sítio).

## Achados na reconfirmação
- /api/table-ss/upload NÃO ignora uma não-mesa: cria row em table_ss_processing_log + gasta Vision -> routing certo importa.
- /api/lobbys/upload ignora não-lobby (log_on_failure=False) -> a direção segura é lobby.
- BUG corrigido: parse_table_ss_filename falhava no token `GGnet.exe` (regex e _FILENAME_SITE_MAP) -> site=None -> caía p/ Vision. Agora normaliza (apara `.exe`, mapeia GGnet->GGPoker). Conserta também as MESAS quando o IT as nomeia com o programa.

## Implementação
- backend/app/routers/table_ss.py: _normalize_site_token() + mapa em minúsculas; regex tolera `.exe`. Ponto único p/ juntar PS/WPN/CoinPoker.
- tools/appimport/app_import.py: classify_it_file() (MESA|LOBBY|SKIP, site, captured_at); process_it_mixed() (pasta única it roteada por nome); dry-run por DEFEITO + flag --ao-vivo; LOBBY_DIR só com --lobby-dir; load_config(). Import.bat sem flag (= dry-run).
- Testes verdes: 906 backend + 8 classificador + 59 table_ss. Dry-run de exemplo: 6 ficheiros -> 3 mesa / 2 lobby / 1 skip, todos certos (incl. GGnet.exe mesa->GGPoker; GGnet.exe lobby->lobby; Shot21 legado->skip).
