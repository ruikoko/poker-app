## Os 4 fixes (`tools/watcher_src/patched_funcs.py`, −126 linhas líquidas)

- **(a) Run intermédia redundante cortada** (`#HRC-REDUNDANT-SECOND-RUN-OLD-CONFIGS`).
  Removido o `start_calculation(ci_target)` + o `set_ci_target_initial` (no-op) do
  `setup_hand`. A 1ª run é lançada pelo **Finish** (spec `WATCHER_FLUXO.md` +
  observação live pt64); o `start_calculation` clicava Calculate outra vez → abria
  um 2º popup Nash → disparava uma run **intermédia** (Full Tree, sem navegar) que
  o HRC **enfileirava** durante a 1ª run (3 runs, 2 janelas Monte Carlo). Agora: a
  seguir ao Finish, espera-se a 1ª run e vai-se **direto** a `navigate` → Selected
  Subtree (**exatamente 2 runs, sem prune**). Órfãs removidas:
  `set_ci_target_initial/refine`, `_set_ci_target_common`, coords `CI_TARGET_FIELD_*`.

- **(b) Run-wait robusto contra o falso-trivial** (`#HRC-RUN-WAIT-FALSE-TRIVIAL`).
  `_wait_for_run_completion`: `timeout_appear` **30s → 180s**; o WARN deixou de
  dizer "run trivial" (passa a "ESPERAVA uma run… nunca é trivial… CONFIRMAR no
  export"). `_find_progress_window_title` aceita agora um **tuple de candidatos** —
  a 1ª run casa `"Hand Setup"` **OU** `"Monte Carlo Sampling"` (robustez ao título
  exacto, que não é verificável só do código).

- **(c') Escrita do CI removida + salvaguarda só-leitura.** Decisão do Rui: o
  default do popup Nash é **sempre 10.0** (= o alvo) → o watcher **não escreve** o
  CI. Removidos `_fill_ci_target_in_popup` (+ chamada), `_find_single_edit`,
  `_read_edit_text` e o caminho pyautogui cego. Salvaguarda nova **só-leitura**
  `_ci_target_readback_warn(10.0)`: lê o "Target CI" da janela de progresso e dá
  **WARN forte se ≠ 10** — zero interação. ⚠️ O formato do título ("Target CI < X",
  de uma foto pt64) **não está verificado em código** → se diferir, falha em
  **silêncio** (nunca falso alarme). Limpeza CB_* morta do Scope
  (`_find_combo_with_item` + consts `CB_GETCOUNT/GETLBTEXT/GETLBTEXTLEN`,
  `WM_SETTEXT/GETTEXT/GETTEXTLENGTH`, `CI_TARGET_POPUP_REL_*`).

- **(d) `select_bounty_mode` hardcoded removido** (`#HRC-BOUNTY-HARDCODED-50PCT`).
  O gate `if is_ko_tournament(prize_path): select_bounty_mode(wpos)` saiu;
  `import_prizes` fica → é o **HRC** que põe o Bounty Mode a partir da estrutura
  importada. (Veredicto pt65: o `select_bounty_mode` corria **depois** do import e
  **esmagava** o modo para PKO 50% cego, só em KO → errado para 0.75/0.40/0.33.)
