## Decisões product (Web/Rui validados)

### Scope (Rui)

- **Aplicar SÓ a 3-bet clássico.** Squeeze fica fora (mantém regra pt42).
- **Opens** ficam como estão (já diferenciados por categoria `SIZES_OPEN_*`).
- **ALLIN como 2ª opção** quando a eff spot-específica dessa posição ≤ 25 BB.

### Regra (CASO A + CASO B)

Para cada posição candidata IP (exclui SB/BB; estes têm `SIZES_3BET_SB_VS_*`
/ `SIZES_3BET_BB_VS_*`):

- **CASO A — posição fez 3-bet na HH real:** 1ª = sizing original;
  2ª = ALLIN se eff spot ≤ 25.
- **CASO B — posição NÃO fez 3-bet (gera sempre, decisão #3 do Web):**
  - eff 0-25 → 2.3 × `opener_to_bb`
  - eff 26-35 → 2.7 × `opener_to_bb`
  - eff ≥ 35 → 3.0 × `opener_to_bb`
  - + ALLIN se eff ≤ 25.

**Eff spot-específica** = `min(opener_remaining, candidate_remaining) / BB`
(snapshot pós-open, no Python; preflop estática é boa aproximação).

### 4 tópicos confirmados (Web)

1. Formato individual `SIZES_3BET_EP/MP/HJ/CO/BU` no template (não dict literal).
2. EP1/EP2 (9-handed) partilham `SIZES_3BET_EP`.
3. CASO B gera mesmo sem 3-bet real (HRC simula vilões com sizings calibrados).
4. Ordem CASO B → CASO A (CASO A sobrescreve sempre na posição do 3-bettor real).
