- **Objectivo:** decidir a profundidade de export que serve o caso de uso
  pré-flop sem fazer ficheiros gigantes.

  viável; ver `GTO_BRAIN.md §8.2/§8.4`).

## Fase 1 (cont.) — diagnósticos v1 + v2 + patch (`94a83b1`)

- **Diagnóstico v1 (estado do patch existente):** até pt34 o patch só abria
  o diálogo Export Strategies e clicava OK sem mexer nos defaults → modo
  "Manual Selection", sem selecção, 1 nó.
- **Diagnóstico v2 (anatomia do combo via Win32):** o combo do diálogo é
  um control nativo. Mudá-lo via `CB_SETCURSEL` (índice 0 = "Manual
  Selection" → índice 1 = "Complete Export") + notificar o pai com
  `CBN_SELCHANGE` + **read-back** do índice seleccionado para confirmar.
- **Patch (`export_strategies`, SWAP em `tools/watcher_src/patched_funcs.py`):**
  1. Muda o combo para "Complete Export" (`CB_SETCURSEL` idx 0→1 +
     `CBN_SELCHANGE` + read-back).
  2. OK no diálogo via `BM_CLICK` (Win32, sem coord — SWT expõe o botão como
     child window nativa, à imagem da cadeia pt30-pt34).
  3. Save As tratado por `_save_as_set_and_click` (clipboard + `BM_CLICK` no
     botão Save, fallback typewrite/Enter).

## Fase 1 (cont.) — descoberta `make_patched_export` (`#DOC-MAKE-PATCHED-EXPORT-OVERRIDES-SWAP`)

- **Sintoma potencial:** um SWAP de `export_strategies` em `patched_funcs.py`
  podia passar o smoke do trampoline e mesmo assim o `.exe` ignorar a
  mudança em produção.
- **Causa raiz (por disassembly do launcher Baltazar
  `hrc_watcher_apr19_launcher.pyc`):** o `main()` do launcher corre
  `g['export_strategies'] = make_patched_export(g)` **depois** do `exec` do
  trampoline (offsets `main()`: 154 exec → 232-256 override → 260
  `g['main']()`). Ou seja, **sobrescreve** qualquer `export_strategies`
  definida via SWAP → o SWAP tem efeito **zero**.
- **Bypass (resolução):** o `wrapper.py` passa a bootar o trampoline
  **directamente** (`exec` → `MAX_CONCURRENT=1` → `g['main']()`) **sem**
  chamar `make_patched_export`, tornando canónica a nossa
  `export_strategies`.
- **Port self-contained:** o `_save_as_set_and_click` foi portado para
  dentro da nossa `export_strategies` (em vez de depender da versão do
  launcher), para que o bypass do `make_patched_export` não retire o Save As
  robusto.
- **Lição registada:** antes de assumir que um SWAP aterra, confirmar que o
  launcher não monkey-patcha a função pós-`exec`. Ver
  `HRC_ANATOMIA_OPERACIONAL.md §8`.
