## Erros descobertos durante o wipe

### Bug crítico — `discord_tags` perdidos no `_insert_hand`

**Sintoma:** Após import dos 3 ZIPs (47k HHs), das 185 placeholders Discord existentes:
- 110 colidiram com HHs reais (esperado) → mas `discord_tags` ficou `[]`
- 75 não colidiram → mas TAMBÉM perderam `discord_tags`
- Resultado: `with_discord_tags` caiu de 185 para **1**

**Root cause (diagnosticado por Claude Code):**
- Schema da coluna `discord_tags`: `ARRAY[]::text[] DEFAULT` (vazio mas **não-NULL**)
- Fix de ontem (commit `823a9f4`) usava `COALESCE(discord_tags, %(placeholder)s)`
- COALESCE só substitui se valor for NULL — array vazio passa intacto
- Resultado: INSERT escrevia `[]` (default), COALESCE preservava `[]`, placeholder perdia-se

**Fix (`7de889f`):**
- `discord_tags = COALESCE(NULLIF(discord_tags, ARRAY[]::text[]), %(discord_tags)s)`
- Mesmo padrão para `hm3_tags` (com `array_remove(..., 'GGDiscord')` para strip do marker)
- `tags` com NULLIF defensivo em ambos os lados do concat aditivo
- Capturado `hm3_tags` no `placeholder_metadata`

**Backfill (`f5dbc16`):**
- Endpoint TEMP `POST /api/discord/admin/backfill-empty-discord-tags`
- Cohort: 74 mãos `origin='discord'` AND `discord_tags=[]` AND `entry_id IS NOT NULL`
- Resolução: `JOIN entries → discord_sync_state → channel_name`
- Resultado: **74/74 reparadas**, zero erros

## Lições / observações

1. **Schema defaults importam tanto como o código:** `ARRAY[]::text[] DEFAULT` não é equivalente a NULL para COALESCE. Padrão NULLIF necessário para fallback funcionar.

2. **Bugs arquitecturais reaparecem em contextos diferentes:** o gap "fluxo X enriquece mãos mas não cria `hand_villains`" apareceu em rematch (ontem), HM3 .bat (ontem), e wipe (hoje). Cada vez precisa de backfill retroactivo. Solução robusta seria trigger DB ou helper único chamado em todos os paths de enrichment.

3. **Validação real só vem com wipe:** o teste end-to-end exposed o bug NULLIF que estava dormente. Sem o wipe, nunca seria descoberto até replicar exactamente o mesmo cenário (placeholder Discord + import HH com mesmo TM).

4. **Webhooks Railway falham silenciosamente:** empty commit é o "fix" universal quando deploy não dispara. Worth keeping em mente.

5. **Per-hand commit > all-or-nothing** para backfills. Permite que edge cases falhem isoladamente sem bloquear o resto do trabalho.
