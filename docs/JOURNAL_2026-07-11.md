## Parte B — Construção da Raiz 2 (aprovada: opção A + ressalva)

**Decisão do Rui:** modelo forte com `entrants` (opção A). Ressalva: `entrants < campo final` é NORMAL

### Resolver (`services/tournament_resolver.py`)
- `_query_summaries(..., return_all=True)` — devolve TODAS as edições da janela (+`total_players`),
  não `LIMIT 1` às cegas. GG + prestart apenas.
- `_disambiguate_editions(...)` — provas DURAS: **H1** nome-exacto · **H2** impossibilidade
  (`entrants > campo·1.05` → exclui) · **H3** não-arrancada-com-eliminações (`anchor < start` E
  `players_left < entrants` → exclui) · **H4** janela de mãos `[1ª−45min, última+120min]` (start do
  TS se não houver mãos). 1 sobrevivente → cola; 2+ com containment estrito único → cola; senão →
  **quarentena** (None).
- `resolve_tournament_number(disambiguate_editions, disambig_anchor_lisbon, disambig_entrants,
  disambig_players_left)` — TIER 0 chama a desambiguação; quarentena via `tier='edition_quarantine'`.
  **Âncora UTC→Lisboa** (o `posted_at` é UTC; o resto Lisboa naive).

### lobby_sync (`services/lobby_sync.py`)
- `_VALID_RESULTS += 'edition_quarantine'`. `process_lobby_message` + `reconcile_lobby_logs` passam
  os params de desambiguação (âncora Lisboa, entrants, players_left) e mapeiam `tier=='edition_quarantine'`
  → `result='edition_quarantine'` (não escreve payout). O reconcile inclui `edition_quarantine` no
  retry → auto-resolve quando chegarem mãos/TS que dêem prova.
