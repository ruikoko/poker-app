### ★ Extensão Trabalho A — multiplicador de efetiva 3-bet (commit `7e38d89`)

Decisão product do Rui: para os 5 buckets de 3-bet clássico
(`SIZES_3BET_IP` / `_BB_VS_SB` / `_BB_VS_OTHER` / `_SB_VS_BB` / `_SB_VS_OTHER`)
o sizing real da HH deixa de ser usado. Em vez disso, aplica-se um
multiplicador ao default do template em função da efectiva. Opens / squeezes
/ 4-bets / 5-bets inalterados.

**Fronteiras escolhidas** (convenção `eff >= threshold`, lower-inclusive
upper-exclusive — justificação no commit message):

| Faixa de efetiva (BB) | Multiplicador | ALLIN como 2º size |
|---|---|---|
| `[35, ∞)` | × 1.00 (defaults intactos, sem override) | sim |
| `[30, 35)` | × 0.90 | sim |
| `[25, 30)` | × 0.80 | sim |
| `[18, 25)` | × 0.70 | sim |
| `< 18` | array `[ALLIN]` só | (single) |

Resolução das fronteiras nos pontos exactos: 25 BB → ×0.80 (não ×0.70);
18 BB → ×0.70 (não ALLIN-só). Documentado no source.

**Estrutura técnica:**
- `_CLASSIC_3BET_DEFAULTS` em `hrc_script_gen.py` regista os 5 defaults
  do template canónico (6 / 10 / 8 / 11 / 8 BB).
- `_classic_3bet_band(eff_bb) -> (mult, shove_only)` resolve a faixa.
- `_compute_classic_3bet_overrides(eff_bb)` produz dict `{var: [size, ALLIN]}`
  para os 5 buckets; vazio se eff ≥ 35; `[ALLIN]` só se eff < 18.
- `build_sizings_overrides` modificado: bc==2 não-squeeze é **ignorado**
  (sizing real da HH descartado para classic 3-bet); post-loop injecta a
  regra do multiplicador.
- Squeezes (bc==2 com callers_before>0) continuam a usar sizing real da HH
  via os `SIZES_3BET_SQUEEZE_*`.

**Tests:** +16 (5 sobre `_classic_3bet_band`, 5 sobre
`_compute_classic_3bet_overrides`, 6 sobre `build_sizings_overrides` em
integração). 2 tests existentes refactorizados para reflectir que classic
